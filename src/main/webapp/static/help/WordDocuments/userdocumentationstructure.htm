User Documentation Structure

The structure of information is organized in such a way as to mirror the software application’s user interface wherever possible.  In order to reduce the required effort for documentation maintenance, the documentation is structured so as to not repeat the same material across various chapters.  Topics are designed to cross-reference appendices to reduce redundant information within procedural topics and give you one common location for information lookup.

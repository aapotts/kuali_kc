Action Taken

An action taken on a document by a Reviewer in response to an Action Request that results in one of the following action statuses:

Acknowledged - Reviewer has viewed and acknowledged document.

Approved - Reviewer has approved the action requested on document.

Blanket Approved - Reviewer has exercised blanket approval up to specified point in route path on document.

Cancelled - Reviewer has canceled document; routing stopped.

Cleared FYI - Reviewer has viewed the document and cleared all of his/her pending FYI(s).

Completed - Reviewer has completed all actions requested on document.

Created Document - User has created a document and initiated its routing process in Kuali Enterprise Workflow.

Disapproved - Reviewer has disapproved document; routing stopped; Acknowledge Requests are sent to prior approvers.

Logged Document - Reviewer has logged a message to the document; namely, he/she has simply added a message to the route log of the document.

Moved Document - Reviewer has moved the document either backward or forward in its routing path.

Returned to Previous Node - Reviewer has returned the document to a designated previous routing node, all the actions taken between the current node and the target return node will be removed, and all the pending requests on the document are deactivated.

Routed Document - Reviewer has started the routing of an INITIALIZED or SAVED document.

Saved - Reviewer has saved document for later consideration.

Superuser Approved Document - Superuser has approved the entire document, disregarding any remaining routing.

Superuser Approved Node - Superuser has approved the document through all nodes up to (but not including) a specified node. When the document finally lands at the specified node, requests will be generated as normal.

Superuser Approved Request - Superuser has approved a single pending Approve or Complete Action Request. The document goes on to next routing node.

Superuser Cancelled - Superuser has canceled the document without the need of a pending Action Request to him/her on the document.

Superuser Disapproved - Superuser has disapproved the document without the need of a pending Action Request to him/her on the document.

Superuser Returned to Previous Node - Superuser has returned the document to a previous routing node without the need of a pending Action Request to him/her on the document.

Users Panel

The Users panel displays each user assigned to that proposal’s development.  It lists the unit and the assigned role for each user:

 

Figure 183 Users Panel

 

KC automatically displays a list of users that have been assigned permissions through the Unit Hierarchy.  An Aggregator may override the Unit Hierarchy for this proposal by adding or deleting a user or changing a user’s role on the Users panel. 

There may be more than one user assigned to the Aggregator role.

For more information on the Unit Hierarchy, contact your KC System Administrator.

 

Table 60 Users Panel Definitions

Field

Description

User Name

Required

The KcPerson Id of a user you are adding to the Permissions list or of the person listed on that row

For more information about adding a user, see  Proposal Development Document, Permissions tab, Procedure for Adding a User.

KC displays the user’s Full Name, the Unit #, and the Unit Name on the Users panel after you select the KcPerson Id.

Full Name

Read only

The user’s first and last name

Unit #

Read only

The user’s Unit Number

Unit Name

Read only

The user’s Unit Name

Role

Required

The role assigned to the user

Click  to select a Role of Aggregator, Narrative Writer, Budget Creator, or Viewer.

A user may be assigned more than one role. 

The exception is the user with the Aggregator role.  It is not necessary to add a role to a user with the Aggregator role because this user already has all permissions for this proposal.

A user can only be assigned to a role once on a proposal.

A proposal must have at least one user with the Aggregator role.  KC will not allow the only Aggregator to remove him- or herself.

Actions

These action buttons are only available to users with the Aggregator role.

 

 Click to add this user to this proposal.

For more information about adding a user, see  Proposal Development Document, Permissions tab, Procedure for Adding a User.

 

 Click to change this user’s role on this proposal.

For more information about editing a user’s role, see  Proposal Development Document, Permissions tab, Procedure for Changing a User’s Role.

 

 Click to delete this user from this proposal.

For more information about deleting a user, see  Proposal Development Document, Permissions tab, Procedure for Deleting a User.

Table 74  Project Personnel Field Descriptions and Procedures

Field

Description

Person

Display-only

 

The specific individual identified as part of the proposal or the To-be-named role. 

Names come from the Key Personnel tab on the Proposal Development Document or by Employee, Non-Employee, or To-be-named lookups. 

Job Code

 

Required

 

Enter the alphanumeric value for the Human Resources position identifier directly associated with the individual named in the Person column.

KC does not require this field before you Add a new personnel line, but does require it before using the  action or moving away from the tab. 

Appointment Type

 

Required

 

From the dropdown list, select the type of employee or duration of the appointment for each person listed. 

Names from the Key Personnel tab do not always have this information. 

Appointment Type drives salary calculations. 

Base Salary

 

Required

 

Enter the individual's current salary less any special pays (administrative supplements).

Some institutions link their HR tables to this field so that the salary auto loads from the system of record (not KC).

Salary Effective Date

 

Required

 

The start date for earnings from the project budget

Also identifies the start date for the individual's annual pay increase.

Salary Effective Date is particularly useful when a person's first day on the project is not the start date of a budget period.

Use the  to locate and select a date, or manually enter the date using the mm/dd/yyyy format.

Adding or deleting entries on the Project Personnel panel has no effect on the contents of the Key Personnel tab on the Proposal Development Document.  If you enter a name on the Key Personnel tab, you must also delete it there. 

You cannot delete a name if expenses for that person remain on the Personnel Detail panel.

Before deleting an Employee, you must first delete the employee's budget details.

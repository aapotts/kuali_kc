You Are Here: Maintenance Document Operations

Overview

KC relies on a variety of user-defined reference tables to function properly.  These tables define attributes the system uses for validating data and providing Lookup values as needed by KC users.  These tables also control the complex relationships between system elements to facilitate internal and external reporting.  KC’s reference tables receive new data and maintenance inputs via electronic inputs called e-docs.  E-Docs are commonly referred to as maintenance documents and are usually maintained from within the central administration of an institution. 

Each of the KC reference tables is maintained by a specific e-doc.  Upon completion, e-docs are routed for approval before any update is applied.  Whereas all users can lookup KC reference table information, access to their maintenance documents is limited to the members of a particular workgroup and only the authorized members of the workgroup may authorize updating of tables.  User permissions control the level of involvement each user may exercise.  While privileged members have the ability to initiate new maintenance documents, other users are limited to searching for the values via the lookup screens.

KC maintenance documents are grouped by related functions on the Maintenance Menu tab.  Each provides a handy, e-doc user interface for creation and maintenance of reference table information that is used by other e-docs in KC.  They allow for lookup and selection of valid values when populating fields to complete other e-docs.  All of the maintenance documents are viewed and maintained in a similar fashion.  Selecting the desired reference table link from the maintenance menu takes you to that maintenance document’s lookup screen.  From that screen you may create a new document by clicking the create new link, or search the table to view, edit, or copy a table value.  These common operations are explained below, using the Proposal Type maintenance e-doc as an example.

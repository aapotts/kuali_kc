Standard Header

The e-doc header appears in the top right corner of all e-docs.  It contains system information about the document:

 

Figure 38 Standard E-Doc Header

 

If this document (e-doc) originated as a copy of another document (e-doc), KC displays that information in the upper-right corner of the header.  Figure 38 shows that document # 2416 resulted from copying document #2314. 

2314 is underlined, indicating that you can click it to display document #2314.

If this document did not start as a copy of another document, then KC does not display the Copied from Document ID columns in the header.

 

Table 6 Standard E-Doc Header Field Definitions

Field

Description

Doc Nbr.

Unique number used to identify each document (e-doc)

When you create a new document, KC automatically assigns it a sequential document number.  KC numbers all documents in the order in which they were created.

KC assigns the document number (also known as the Document ID) when the document is created. 

Initiator

Identifies the user ID of the person who created this document (e-doc)

If the user ID is underlined, you can click it to get additional information about the user.

Sponsor Name

An identification (up to 6 alphanumeric characters) for the organization or agency that is providing support for this sponsored project

This comes from the Sponsor ID field on the Required Fields for Saving Document panel on the Proposal tab of the Proposal Development Document.

Status

The document’s completion status

This is not the same as the document status in workflow.

Created

Date-time stamp

Tells the time and date of document creation

PI

 

Principal investigator) is the person who has primary responsibility for all essential aspects of the project or protocol.  The PI/Project Director/Program Director (PI/PD) is the individual designated by the grantee and approved by the sponsoring agency who is responsible for the scientific or technical direction of the project.

Conditionally Displayed Fields in the Document Header

Copied from Document ID

ID number of the document copied when creating this document

Correct Document ID

KC lets you reverse a fully approved document transaction with the Error Correction function.  When one document is a correction of another, Kc displays the number of the original document here. 

KC displays this field only when this document was created from an existing document using the Error Correction feature.

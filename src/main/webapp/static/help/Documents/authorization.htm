Authorization

Your KC role determines your authorization to create, modify, merge, submit, view, or print a proposal log.  Your institution may create its own user roles and authorizations, or you may use the default KC roles and authorizations:

 

Table 96 KC Default Authorizations for Proposal Log

User Class

Role

Authorizations

OSP Administrator

Maintainer

The OSP Administrator can create, modify, merge, submit, and print a proposal log.

OSP Administrator – Contractor

Maintainer

The OSP Administrator can create, modify, merge, submit, and print a proposal log. 

Unit/Dept Administrator

Viewer

A Unit and a Dept Administrator can view and print a proposal log

PI/Co-PI/Key Personnel

Viewer

The PI, Co-PI, and Key Personnel can view and print a proposal log.

The PI, Co-PI, and Key Personnel can request a proposal number and view their proposals in the Proposal Log; they need to request changes through the appropriate mechanism and route them to the OSP Administrator.

 

For the procedure to merge a new, permanent Proposal Log with an existing temporary Proposal Log, see below:  Proposal Log Document, Procedure for Merging a new Proposal Log with an Existing Temporary Proposal Log.

For the procedure to merge a temporary Proposal Log with an existing Institutional Proposal, see below:  Proposal Log Document, Procedure for Merging a Temporary Proposal Log with an Institutional Proposal.

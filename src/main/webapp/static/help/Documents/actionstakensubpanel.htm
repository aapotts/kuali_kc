Actions Taken Sub-Panel

The Actions Taken sub-panel lists the actions taken in previous workflow nodes:

 

Figure 427 Actions Taken Sub-Panel

 

Table 160 Actions Taken Sub-Panel Definition

Field

Description

Action

Read only

 

The status of the action and the Action Request type.  The Action Request type displays below the status.

Action statuses are:

PENDING

COMPLETE

SAVED

Action Request Types are

APPROVE: This action is pending approval.

ACKNOWLEDGE: This action is pending acknowledgement.

FYI: This action is pending FYI.

Taken By

Link

 

The name of the person who completed this workflow action

Click on the person’s name link to see details about this person.

Delegator

Read  only

 

The name of the person who delegated this action

For more information about the Delegator, see  Kuali Enterprise Workflow, Workflow Example or contact your KC System Administrator.

Time/Date

Read only

 

The date and time this action was taken

Annotation

Read only

 

Notes from users who made changes to the document

 

Users with appropriate permissions can make changes to the document during the workflow.  Annotations enable them to make note of the changes.  The note is displayed in the Annotation field of the Route Log.

KC can generate an annotation.  An example is when someone blanket approves a document.  KC adds a message about the blanket approval in the Annotation field.

Object Code Name

Required

 

Click  in the Object Code Name column.  

Select the appropriate object code name from the list.

Click  to see the list of object code names.  The Lookup list is identical to the drop–down list.

Your institution manages the list of Object Code Names.

Identifying personnel before adding details includes identifying the Object Code Name defining their relationship to the project. 

For each separate Object Code Name identified among the project personnel, KC creates a sub-panel on the Add Details section. 

In Figure 244, you can see that at least two Object Code Names are in use because the associated sub-panels are visible:

Faculty Salaries Non-Tenured - On

Research Asst - On 

Each Object Code Name has a secondary identifier in its name: On or Off.  This secondary designation sub-divides the Object Code Names by identifying whether the position is On campus or Off campus.

Your institution manages the list of object code names.

 

Object Code Names are not the same as budget categories.

Group

Conditionally Required

 

The Group designation is an additional identifier than can be used to sub-divide object codes or link personnel spanning multiple object codes.  When the Group identifier is used, the sub-panels header includes the Group.

Example: Six individuals have the Object Code Name Administrative Staff.  All work On campus.  When there is no Group, all six names appear on the Administrative Staff - On sub-panel.  If three of the six names receive a Group of Admin-A, then KC lists those three under a separate sub-panel headed with Administrative Staff - On/Admin-A.

Action

Click  to complete the process of identifying the Object Code Name and adding an individual or To-be-named role to the Personnel Detail panel.

 

After an individual is added, the name or role displays on the sub-panel identified by the Object Code Name/Group to which they are assigned.  For reference purposes, call these sub-panels Object Code Name sub-panels.

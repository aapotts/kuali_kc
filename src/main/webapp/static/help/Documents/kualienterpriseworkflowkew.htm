Kuali Enterprise Workflow (KEW)

 

Kuali Enterprise Workflow (KEW) is a rules-based electronic routing workflow engine.  It automates the routing of electronic documents (e-docs) to individuals and workgroups for approval based on your institution’s business rules. 

 

When an e-doc is complete, you use Workflow Action Buttons to submit it to KEW workflow routing.  After you submit the e-doc, KC generates an Action Request to persons and workgroups that are required to approve the document. 

Approvers use their Action List to view the Action Request and open the document to review it.  As an approver, you can use Preferences to change the way KC displays your Action List.  In Preferences, you can specify how often you want KC to notify you about new Action Requests.

After reviewing the document, you use the Workflow Action Buttons to approve it.  You can use Workflow Action Buttons to run a system-to-system (S2S) submission to www.grants.gov for eligible Proposal Development documents.

The Route Log displays the document’s Route Status.

You can locate documents using Doc Search. 

You can add persons or workgroups to KEW workflow with Ad Hoc Routing.

 

For information about your institution’s specific KEW rules, contact your KC System Administrator 

For technical information about KEW, see the Online KEW User Guide

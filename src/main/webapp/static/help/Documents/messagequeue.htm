Message Queue

The Message Queue workflow screen allows administrators to monitor messages that are flowing through the Service Bus.  Messages can be edited, deleted or forwarded to other machines for processing from this screen.

 

Figure 1037 Message Queue Workflow Screen Example

 

Table 756 Message Queue Workflow Screen – Field Descriptions

Field

Description

Current Node Info

Display-only.  A consolidated view of current identifying, status and statistical information is displayed about the current node.

Filter

Click the filter button to filter the route queue based on criteria entered (machine’s service namespace or IP address) in the filter section fields, and queues them up for subsequent processing.

Execute Message Fetcher

Click this to get queued messages based on the filter criteria for document services currently in the route queue.

Message Queue Id

Displays the unique numeric value for each message in the route queue.

Service Name

The name of the service is displayed (for example, those pertaining to document routing or processing) preceded by the associated service namespace in brackets.

Service Namespace

The uppercased unique name of the service namespace is displayed.

IP Number

The Internet Protocol numeric value is displayed with decimal points.

Queue Status

The queue status of the message is displayed, which is one of the following: QUEUED, ROUTING, OR EXCEPTION.

Queue Priority

The numeric priority value assigned to the message by the system is displayed.

Queue Date

The time and date the message entered the queue.

Expiration Date

The time and date the message expires.

Retry Count

The number of times the sending of the message has been attempted.

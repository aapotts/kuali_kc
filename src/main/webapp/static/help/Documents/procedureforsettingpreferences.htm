Procedure for Setting Preferences

You set up your preferences by filling in the fields on the Workflow Preferences screen.  You can change your preferences at any time.  Some preferences do not go into effect until you log into KC again. 

 

1.  Click Preference in the Workflow menu group or  on the Action List screen.

•        

•         Workflow Preferences screen:

•

Preference

Description

Automatic Refresh Rate

Optional

 

The frequency in minutes you want your Action List to refresh

Enter a number in whole minutes.  If you enter 0, your Action List will not refresh automatically. You can click the refresh button on the action list page to refresh manually.

Action List Page Size

Optional

 

The number of action requests you want KC to display on each page

Enter a number of action requests for KC to display per page.

Email Notification

Optional

 

The frequency you want KC to email you notifications about your workflow action requests:

None: You do not want KC to email you any of your workflow action requests

Daily: You want KC to email you once a day with your workflow action requests for that day

Weekly: You want KC to email you once a week with your workflow action requests for that week

Immediate: You want KC to email you immediately with each workflow action request

Click  to select an Email Notification of None, Daily, Weekly or Immediate.

Receive Primary Delegate Emails

Optional

 

Your indication of whether you want to receive email notifications about workflow action requests for which you are the primary delegate

Click  to receive email notifications about workflow action requests for which you are the primary delegate.

For more information about Primary Delegates, see  Workflow Example or contact your KC System Administrator.

Receive Secondary Delegate Emails

Optional

 

Your indication of whether you want to receive email notifications about workflow action requests for which you are the secondary delegate

Click  to receive email notifications about workflow action requests for which you are the secondary delegate.

For more information about Secondary Delegates, see  Workflow Example or contact your KC System Administrator.

Delegator Filter

Optional

 

Your indication of whether you want action requests for which you are a secondary delegate to display in your Action List:

Secondary Delegators on Action List Page:  You want action requests for which you are a secondary delegate to display on your Action List.

Secondary Delegators only on Filter Page: You do not want action requests for which you are a secondary delegate to display on your Action List.  If you select this option, you can only see your secondary delegate action requests in on a Filter Page

Click  to select Secondary Delegators on Action List Page or Secondary Delegators only on Filter Page.

For more information about Secondary Delegates, see  Workflow Example or contact your KC System Administrator.

Primary Delegates Filter

Optional

 

Your indication of whether you want action requests for which you are a primary delegate to display in your Action List:

Primary Delegates on Action List Page:  You want action requests for which you are a primary delegate to display on your Action List. Primary Delegates only on Filter Page: You do not want action requests for which you are a primary delegate to display on your Action List.  If you select this option, you can only see your primary delegate action requests in on a Filter Page

Click  to select Primary Delegates on Action List Page or Primary Delegates only on Filter Page.

For more information about Primary Delegates, see  Workflow Example or contact your KC System Administrator.

Fields Displayed in Action List

Optional

 

Your indication of whether you want the listed field to display as a column in your Action List

Use Outbox does not display as a column in your Action List.  Instead, it displays as a link.

Available fields are:

Document Type: The document type.  Examples are Proposal Development document and Budget document.

Title: The document title

Action Requested: The action you are requested to perform on this document

Initiator: The name of the user who created the document

Delegator: The person who delegated the action request to you

Date Created: The date and time the document was created

Date Approved: The date and time the document was approved

Current Route Node(s): The current route node; examples of route nodes are INITIATED, PENDING APPROVE and COMPLETED.

Workgroup Request: The name of the workgroup; you may want to view only workgroups you belong to.

Document Route Status: The KC KEW workflow status of the document

Application Document Status: The document status

Clear FYI: Check if you want the ability to clear the FYI from the Action List without opening the document.

Use Outbox: Display the Outbox link on your Action List page

Click  next to each field you want to display as a column in your Action List. 

If you click  next to Use Outbox, Outbox will display on your Action List page.

For more information about the KEW Outbox, see  Kuali Enterprise Workflow, Action List, Outbox.

Document Route Status Colors for Action List Entries

Optional

 

The color you want documents of each route status to display in your Action List

Click  corresponding to the color for each document route status.

The colors will take effect in your Action List the next time you log into KC.

 

Click  to save your preferences.

Click  to reset all preferences to their KC default values.

Click  to cancel your selections and return to the previous screen.
